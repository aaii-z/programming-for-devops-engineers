Step 1: Our First Lookup - A Simple Spell!

"Alright, future Go Gopher!" Froggy cheers, adjusting his tiny coding glasses. "For our very first spell with Go, we're going to teach our DNS Frog Gopher to find the secret address (the IP address) of a famous website. Think of it as finding the exact lily pad where 'google.com' lives! We'll start simple and tell our program the website name directly in the code."

üê∏ Froggy's Setup Action: Create Your Spellbook!

Let's get our spellbook ready! If you haven't already, create a new folder on your computer named dnsfrog. Inside this folder, create a new, empty text file and name it dnsfrog.go. Froggy reminds you, "All Go magic is written in files ending with .go!"

Key Go Spells: The Bare Bones of a Program

Froggy says, "Every Go spell, no matter how simple or complex, needs a few basic parts to work. Think of it like a recipe!"

  • package main: This is like declaring, "This is my main spellbook!" It tells Go that this code will become a program we can run directly. All programs start in package main.
  • import (...): "No gopher is an island!" Froggy quips. Go has lots of built-in 'packages' (collections of ready-made spells). The import keyword lets us bring in the ones we need. For our first spell, we'll need:
    • "fmt": Froggy calls this the 'Formatting Megaphone'. It helps us print messages to your screen.
    • "log": If our spell fizzles, this package helps print out what went wrong. Very important for debugging!
    • "net": This is our book of networking spells! It has the magic to look up website addresses.
  • func main() { ... }: This is the most important spell in our main package. It's where our program officially starts. Froggy says, "The main function is where the magic begins!" All the code for our program's main task goes between the curly braces { }.

‚ú® Our First Go Spell: The Code!

"Okay, get your fingers ready!" Froggy encourages. "Carefully type (or copy) this magic into your dnsfrog.go file. Don't worry, I'll explain what each part does!"

package main

import (
    "fmt"
    "log"
    "net"
    "os" // Keep os for consistency with Step 2, though not strictly needed for hardcoded Step 1
)

func main() {
    domain := "google.com" // Our practice domain!

    fmt.Printf("Froggy is preparing to look up A records for: %s\n", domain)

    ips, err := net.LookupHost(domain) // The actual DNS lookup!
    if err != nil {
        // Uh oh, something went wrong!
        log.Fatalf("Ribbit! Froggy stumbled! Failed to look up %s: %v\n", domain, err)
    }

    if len(ips) == 0 {
        fmt.Printf("Hmm, Froggy couldn't find any A records for %s.\n", domain)
        os.Exit(0) // Exit cleanly if no records found
    }

    fmt.Printf("Success! Froggy found these IP addresses for %s:\n", domain)
    for _, ip := range ips { // Loop through each IP found
        fmt.Printf("  - %s\n", ip)
    }
}

üê∏ Froggy Breaks It Down:

Let's see what those lines of code mean:

  • domain := "google.com": Froggy says, "Here, we're creating a 'variable' named domain and giving it the value "google.com". Variables are like little boxes where we can store information. The := is a shortcut in Go to declare and assign a variable at the same time!"
  • fmt.Printf(...): "This is our Formatting Megaphone in action! It prints messages to your screen. The %s is a placeholder that gets replaced by the value of domain. The \n means 'start a new line'."
  • ips, err := net.LookupHost(domain): "This is the core magic! We're calling the LookupHost spell from our net spellbook. It tries to find the IP addresses for the domain." Froggy adds, "This spell actually gives back *two* things: ips (a list of IP addresses it found) and err (any error that happened, or nothing if it worked!)."
  • if err != nil { ... }: "Super important, this one!" Froggy emphasizes. "It checks if our err box has something in it (!= nil means 'not nothing'). If there's an error, we use log.Fatalf(...) to print a helpful error message and stop the program. Good gophers always check for errors!"
  • if len(ips) == 0 { ... }: "Sometimes a domain exists but has no 'A' records. This checks if our list of ips is empty. If so, we print a message and exit."
  • for _, ip := range ips { ... }: "If we found IPs, this 'for loop' hops through each ip in our ips list and prints it out. The _ means we don't need the index number for this loop."

üê∏ Froggy's Showtime: Running Your Spell!

Save your dnsfrog.go file. Now, open your computer's command line (terminal), go into your dnsfrog folder, and type this command:

go run dnsfrog.go

Froggy cheers, "If everything is correct, you should see the IP addresses for google.com! You did it! That's your first working Go program!"

Froggy's First Go Tip: Go is all about simplicity and clarity! Notice how clean the error checking is. And go run is a super handy command to quickly compile and run your Go programs without creating a separate executable file yet. It's great for testing your spells as you write them!
