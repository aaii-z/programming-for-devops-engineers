Step 1: Our First Lookup - A Simple Spell!

"Alright, let's cast our first Go spell!" Froggy exclaims. "We'll make a tiny program that looks up the IP address (an 'A' record) for a website. For now, we'll just hardcode 'google.com' to see how it works."

First, create a new directory for our project, maybe called dnsfrog. Inside this directory, create a file named dnsfrog.go. Froggy says, "All Go source code files end with .go!"

üê∏ Froggy Explains: The Basic Structure of a Go Program

Every Go program needs a few key ingredients:

  • package main: This tells Go that our program is an executable one (a command we can run), not just a library of spells for other programs to use. Our main spellbook starts here!
  • import (...): Go has a treasure trove of built-in "packages" that give us ready-made spells. The import statement is how we tell Go which spellbooks (packages) we need. For this step, we'll need:
    • "fmt": Short for "format", this package helps us print things to the screen. It's like Froggy's magic megaphone!
    • "log": This package helps us print error messages if something goes a bit wobbly.
    • "net": This package is full of networking spells, including the ones for DNS lookups!
  • func main() { ... }: This is the main entry point of our program ‚Äì the first spell Go will cast when we run our tool. All our main logic will go inside these curly braces {}.

‚ú® The Code for Our First Lookup

Type (or copy) this into your dnsfrog.go file, Froggy cheers you on!

package main

import (
    "fmt"
    "log"
    "net"
)

func main() {
    domain := "google.com" // Our hardcoded domain for now!

    fmt.Printf("Froggy is looking up A records for: %s\n", domain)

    ips, err := net.LookupHost(domain) // This is the magic DNS lookup spell!
    if err != nil {
        log.Fatalf("Oh no, Froggy couldn't look up the domain %s: %v\n", domain, err)
    }

    if len(ips) == 0 {
        fmt.Printf("Froggy found no A records for %s.\n", domain)
        return
    }

    fmt.Printf("Hoppy day! Froggy found these IP addresses for %s:\n", domain)
    for _, ip := range ips { // We loop through all the IPs found
        fmt.Printf("- %s\n", ip)
    }
}

Running Your First Go Spell!

Froggy says, "Now for the exciting part!" Open your terminal, navigate to your dnsfrog directory, and type:

go run dnsfrog.go

If all goes well, you should see the IP addresses for google.com! "Ribbiting, isn't it?" Froggy exclaims. "You've just performed your first DNS lookup with Go!"

What happened here?

  • We declared a domain variable and set it to "google.com".
  • net.LookupHost(domain) is the Go function that does the actual work of finding IP addresses (A or AAAA records) for the domain.
  • It returns two things: a list (called a "slice" in Go) of IP addresses (ips) and a potential error (err).
  • if err != nil { ... } is how we check for errors in Go. If err is not nil (nil means 'nothing' or 'no error here'), we use log.Fatalf(...) to print an error message and stop the program. Froggy says, "Good wizards always check for misfires!"
  • If there's no error, we loop through the ips slice using a for _, ip := range ips loop and print each IP address.

Froggy's First Go Tip: Go is all about simplicity and clarity! Notice how clean the error checking is. And go run is a super handy command to quickly compile and run your Go programs without creating a separate executable file yet.
