Step 2: Taking User Input - Making it Interactive!

"Hardcoding 'google.com' was fun for a first spell," Froggy says, "but a real DNS tool needs to look up any domain we tell it to! Let's make our DNS Frog Gopher interactive by reading the domain name from the command line."

üê∏ Froggy Explains: Command-Line Arguments with os.Args

When you run a command in your terminal, like go run dnsfrog.go example.com, the words that come after the command itself (example.com in this case) are called command-line arguments.

Go gives us a way to access these arguments through a special variable called os.Args. To use it, we first need to import the "os" package, which provides functions for interacting with the operating system.

os.Args is a "slice" of strings. Froggy explains, "Slices are like flexible, expandable lily pads that can hold a list of similar items ‚Äì in this case, strings!"

  • os.Args[0] is always the name of the program being run (e.g., dnsfrog.go if you use go run, or just dnsfrog if you run a compiled binary).
  • os.Args[1] is the first argument after the program name.
  • os.Args[2] is the second, and so on.

We also need to check if the user actually provided an argument. If they don't, os.Args will only have one item (the program name), and trying to access os.Args[1] would cause a "panic" (Go's way of saying something went very wrong!). Froggy says, "Always check your spells before you cast them too widely!" We can check the length of the slice using len(os.Args).

‚ú® Updating Our Code for User Input

Let's modify our dnsfrog.go file. Froggy will point out the changes:

package main

import (
    "fmt"
    "log"
    "net"
    "os" // <-- Froggy says: New import!
)

func main() {
    // Froggy says: Check if the user gave us a domain name!
    if len(os.Args) < 2 {
        // fmt.Println("Usage: go run dnsfrog.go ")
        // log.Fatalf("Oops! Froggy needs a domain name to look up. For example: go run dnsfrog.go google.com")
        // Better: Print usage to stderr and exit with a different status for scripts
        fmt.Fprintf(os.Stderr, "Usage: %s \n", os.Args[0])
        fmt.Fprintln(os.Stderr, "Oops! Froggy needs a domain name to look up.")
        fmt.Fprintln(os.Stderr, "For example: go run dnsfrog.go google.com")
        os.Exit(1) // Exit with status 1 to indicate an error
    }

    domain := os.Args[1] // Get the domain from the first argument

    fmt.Printf("Froggy is looking up A records for: %s\n", domain)

    ips, err := net.LookupHost(domain)
    if err != nil {
        log.Fatalf("Oh no, Froggy couldn't look up the domain %s: %v\n", domain, err)
    }

    if len(ips) == 0 {
        fmt.Printf("Froggy found no A records for %s.\n", domain)
        return
    }

    fmt.Printf("Hoppy day! Froggy found these IP addresses for %s:\n", domain)
    for _, ip := range ips {
        fmt.Printf("- %s\n", ip)
    }
}

üß™ Testing Our Interactive Tool

Save your dnsfrog.go file. Now, in your terminal, try running it with different domain names:

go run dnsfrog.go developers.google.com
go run dnsfrog.go gnu.org
go run dnsfrog.go yourfavoriteblog.com

Froggy asks, "What happens if you run it without any domain name?"

go run dnsfrog.go

You should see our helpful usage message! "See? Our error check worked!" Froggy ribbits proudly.
